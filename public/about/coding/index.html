Coding

The technology has two modes of operation:
AUTOMATIC: Interacts with the energies of the environment, generating free quantum particles.
FOCUSED: The user uses a code to direct the resonance towards a specific end.

The use of codes is a mental exercise that does not require additional technology.
The user only needs to link the concepts of the code with the problem they are trying to solve.
The more broad the thought behind the code, the more chances of seeing an effect.

To build a code it is necessary to link the concepts of the scheme forming a closed chain.
Connection lines must be respected since not all concepts are related to each other.
The codes can be applied both to the user of the device and to another person with whom they have a close mental bond.

When the user sends a code, a resonance is generated between the receiving biofield and the message transmitted by the signal.
The response is usually perceived as a 'butterfly effect' and not as the acceptance of an order

FUNCTION


Represents the action we want to achieve with the code.

Like a mathematical equation, it depends on other concepts to complete the operands and obtain the result.

TRANSFORMATION


It is applied when it is necessary to change some element of the situation.

Unlike work, it is a natural process that is triggered by the individual's own free will.

REACTION


Each modification induced by the codes generates a percentage of rejection by the biofield.

Including the concept of reaction in the code reduces this rejection and increases the chances of success.

NEO


They are the products that result from new combinations with the environment.

It is applied when problems cannot be solved by modifying existing elements. For example, when seeking to generate new ideas.

WORK


Represents the physical or mental work that the individual must perform to achieve the objective.

It is related to the transformation of ideas into forces capable of modifying the environment.

MEMORY


It is responsible for storing physical (fat) and abstract (experiences) elements within the body.

It needs to be included in your code so that it remains effective without the need to implement it on a daily basis.
